# ポーカーレンジ構築・評価システム仕様書

## 1. 概要

本システムは、

1. ハンドレンジを構築するツール
2. 構築したレンジを評価するソルバー

を分離して実装する。

目的は、特定のポジション、ボード、プレーラインに対するレンジ構築とEV評価を行うことである。

---

# 2. システム構成

システムは以下の2コンポーネントから構成される。

## 2.1 レンジ構築ツール

### 役割

- ハンドレンジの編集
- レンジの可視化
- 各プレーラインごとのレンジ保存

### 機能

- ポジション選択
- ボード選択
- プレーライン選択
- ハンドの追加・削除
- レンジの保存
- レンジの読み込み

### UI

13×13のハンドマトリクスを表示する。

例:

| AA | AKs | AQs | ... |
|----|-----|-----|-----|
| AKo | KK | KQs | ... |
| AQo | KQo | QQ | ... |

### ハンド操作

- セルクリックで選択
- 選択済みハンドを次ストリートへ引き継ぐ
- 未選択ハンドは削除扱いとする

---

## 2.2 ソルバー

### 役割

保存されたレンジを評価する。

### 入力

レンジ構築ツールが出力したJSONファイル。

### 出力

- Hero EV
- Villain EV
- ナッツアドバンテージ
- レンジアドバンテージ
- バリュー比率
- ブラフ比率
- 推奨アクション

# 4. データ構造

## 4.1 ファイル形式

JSON形式を採用する。

## 4.2 サンプル

```json
{
  "position": "BTN_vs_BB",

  "board": "As5dTc6h8c",

  "line": [
    "flop_b33",
    "turn_x",
    "river_b60"
  ],

  "hero_range": {
    "AA": 1.0,
    "AKs": 1.0,
    "AQs": 0.75,
    "KQs": 0.5
  },

  "villain_range": {
    "ATs": 1.0,
    "88": 1.0,
    "65s": 0.5
  }
}
```

# 5. レンジ管理

## 5.1 レンジ表現

各ハンドは頻度を持つ。

```text
0.0 = 常に除外
0.5 = 半分の頻度で採用
1.0 = 常に採用
```

例:

```json
{
    "AKs": 1.0,
    "AQs": 0.5,
    "76s": 0.25
}
```

---

# 6. プレーライン

プレーラインは各ストリートのアクション列で定義する。

例:

```text
Flop: Bet33
Turn: Check
River: Bet60
```

ファイル名:

```text
flop_b33_turn_x_river_b60.json
```

---

# 7. レンジ遷移

各ストリートで前ストリートのレンジを引き継ぐ。

例:

```text
Preflop Range
      ↓
Flop Bet Range
      ↓
Turn Check Range
      ↓
River Bet Range
```

プレイヤーがハンドを選択すると、

- 選択されたハンドのみ次ストリートへ進む
- 未選択ハンドは除外される
